Total part

Crypt programming languages allow you to assign hotkeys to any operations, and you can live with it until there are too many of them. Then the memory begins to fail, and much of what has been done before is unavailable, so it is easier to do with your hands than to delve into old stocks. Hence the conclusion: you need a repository that allows you to both assign hotkeys to scripts and quickly access the accumulated from the menu. To do this, we intend to use CLCL, a popular clipboard manager, thanks to which we can work with the history of the clipboard, quickly transforming and inserting the saved in it.

The mechanism of work of the program is simple. CLCL has the ability to save templates for inserting the most frequently used fragments (the name used is used in the version used Samples). We enter the script in the template and mark with the heading that this is a script, and not that it should simply be inserted into the text. The program has the ability to assign a combination to the text box (by default Ctrl + V), into which we enter our hot key Ctrl+Shift+F11, according to which the program creates a temporary script and applies it to the selected text, clipboard or its history. If the template has a simple text or picture, they are processed in the usual way.

Hotkeys are assigned to templates from their context menu in the main program window. To quickly call the menu of templates we use a keyboard shortcut Alt + Q, prescribed in Options → Action main window.

The AHK \ CLCL.ahk script file is available for editing (works from the local version of Autohotkey. At the beginning, you can find a number of settings.

There are hot keys Alt+1...Alt+9, which are used to insert the last nine versions of the buffer. With a second clamping Alt, the clipboard is changed without pasting! It works with the standard menu call combination Alt + C.

Syntax and examples

Suppose we need the ability to paste the current date and time into the text. First, it should be noted that this is a script. For this, the heading says ###AHK###, meaning this is an Autohotkey script with an extension ahk (in the case of AutoIt would be ###AU3###). You can limit yourself #ahk#, but scripts will sometimes fall into the history of the clipboard, and so they will be more visible. Now we write a srypt:

###AHK###
SendInput % A_DD "." A_MM "." A_YYYY "  " A_Hour ":" A_Min

And, in fact, with this all - hang a hot key and use it. (When assigning a hot key, be sure to check the nut Paste!)

This is the simplest option, which does not involve interaction with the clipboard and selected text. If there was a selection, it will be overwritten.

But, suppose we want to copy the old text, delete it and insert a new one in its place, for example, by changing its register or adding framing. To change the register, it will look like this:

xxxxxxxxxxxxx
###-AHK###
text:=Clipboard
StringUpper Clipboard, text
Send ^{vk56}

In this case - in ###-AHK### means that the selected text is copied and deleted by program funds. Similarly ###+AHK### will mean that the text is copied, but not deleted, which is necessary, for example, when generating text that will be inserted elsewhere or in another application. Among other things, this allows you not to clog the buffer unnecessarily without copying the text where it is not needed.

You should not insert the text copy operation into the template without the need to paste, since then it will be impossible to process and paste text from the clipboard history menu!

In addition to prefixes in the form of a plus or minus, you can use the endings that are needed in the case of battles (or Python scripts) and emerging console windows. For example, a pattern of the species

xxxxxxxxxxxxx
###BAT=###
gpedit.msc

will call the group policy editor without showing the console window at all, unnecessary here. If replace equal to minus ###BAT-###, the console window will be minimized and you can look into it if necessary by adding pause at the end.

Virtual script and function library

When converting text, it is generated in the AHK folder, where the executable Autohotkey.exe and AutoIt.exe files are located, and has the form temp.ahk, temp.au3, temp.bat, depending on what you use. The extension is determined by the heading, the encoding will be discussed below. The file lives only until the next use of the templates, and if in doubt, you can look in and see what is in it. If the script is hanging due to errors, in the case of Autohotkey and AutoIt, it should be unloaded from the track manually. The Autohotkey library (local) is located in the virtual script folder, the functions used should be in it.

Using text scripts to process text

The CLCL template editor is miserable, and does not even allow you to reliably record changes. You can’t write much there, so the question arises using scripts in third-party text files, which are much more convenient for editing. In this case, the file path is written in the heading, so something like ###D:\MyScript.ahk### or ###Scripts\MyScript.ahk### in the case of using relative paths. Prefixes and endings work in the same way as before.

You can add command line parameters sent to the script by enclosing them in the second block inside the grid, for example ###Scripts\MyScript.ahk###/a /b /c###. Parameters are arbitrary and transmitted as is.

Text scripts are executed on site, without copying them into "virtual" in the AHK folder, so the use of library functions must also be decided "on the spot".

The Markdown section provides examples of using a single script with different parameters for quickly framing text.

Working with clipboard history

Subject to the conditions described above (copy only by means of the program), it is possible to process menu items of the clipboard history. To do this, click with the middle mouse button on the selected menu item called by the combination Alt + C. Another option is to select the required item with cursor keys and press Ctrl or Alt. When working with the history menu, all prefixes in the template header are ignored!

Features of processing very large text files

It should be remembered that:

  1. Strict functions (StrReplace, etc.) work faster associated with regular expressions (RegExMatch, RegExReplace).
  2. Before inserting the text, a delay should be made (100-200 ms or more), otherwise it may simply not work.
  3. Many editors do not swallow the box of dozens of megabytes of text at all. Often this applies to editors with their own tracking clipboard history. From the popular benchmark in this area is AkelPad.
  4. The history file with such manipulations increases dramatically in size. CLCL.ahk has the option to delete it at start or exit.

Capture clipboard and selected text

The "Capture" section of the template menu contains various work options for the TextCapture.ahk script located in the AHK folder. There you can see the script settings and change them, for example, in the separator settings and limiting the number of blank lines in the copied text. When you start, a minimized window appears in the taskbar, in which you can see the captured fragments. You can remove the checkers from the unnecessary and place them up and down. As long as there is a window, you can supplement its contents. There are buttons in the window Copy FIFO and Copy FILO, corresponding to the direct and reverse output of fragments into the buffer. In most cases, there is no need to use this, just run the script manually or using a hotkey, copy fragments, and then paste them using keyboard shortcuts (hereinafter the prefix R means “right”: RCtrl - Right Ctrl and so on):

  • RCtrl + V - insert in a straight sequence
  • RCtrl + RShift + V - in reverse order

The script window will close and it will be unloaded from memory. Captured stories are saved in the Logs subfolder of the script in text files with timestamps.

Escape minimizes the active script window. By combination Shift + Space you can call it and hide it back.

When copying, duplicates are excluded immediately, provided that the option in the script is enabled, this can be judged by the tultip in the center of the screen and the presence or absence of sound.

The script has a sep_list variable that looks something like this: \r\n|\r\n\r\n|\r\n\r\n\r\n|\r\n=======================\r\n|\r\n\r\n\t\t\t--- \i ---\r\n|\r\n\r\n### [\h](\u)\r\n| <...>
It represents a set of fragment separators available in the selection of a script window separated by a vertical line from the drop-down menu. The first value (here - new line) is the default value. In the syntax, it is possible to use the usual designations \r, \n and \t corresponding to the line translation, new line and tabulation.

In addition, the use of the following characters is possible:

  • \i - serial number of the fragment for subheadings, etc.
  • \h is the name of the window from the title. Can be inserted as it is, in the script settings there are options for converting the full path to the title from the file name with or without an extension, as well as trimming of the tailings with the name of the application of the form "- Mozilla Firefox" (do not always work correctly due to the variety of names, so it is turned off)
  • \u - page address, if available. Copying is done in hidden mode, without using hotkeys and clipboard, so as not to clog the history. In Vivaldi, Pale Moon and some other browsers with modified engines, this is not possible.

The first separator is always skipped, unless it contains these three characters, allowing you to consider it a heading. Initial spaces and craps in the final text are deleted automatically.

To select the type of capture and separator, the following command line parameters are used:

  • -2, -3, -4 ... - sets the separator number in the drop-down menu (sep_list variable)
  • -s - capture of selected text
  • -k text capture only by keyboard shortcut
  • -m - capture of multi-page documents in the browser

By default, the clipboard is captured.

Capture selected text

It is carried out on the same principles and with the advent of the same window of fragments. The script reacts to:

  • Highlighting with the mouse, including a double and triple click
  • Left-click scroll
  • Combinations Shift + Left, Shift + Right, Shift + Down, Shift + Up, Shift + PgDn, Shift + PgUp, Shift + Home, Shift + End
  • Combination Ctrl + A

In the script, you can specify the minimum size of the fragment to exclude random emissions (states 20). The usual buffer grip is not turned off, but since copying is preceded by selection, in this case a duplicate message will appear.

Keyboard Capture

It is carried out only by pressing keyboard shortcuts RCtrl + C, Ctrl + Insert, Ctrl + NumLockIns ("gray" Ins on the right side of the keyboard). Regular copying to the buffer and selection are ignored.

Capture multi-page documents

Designed for quickly copying multi-page documents in the browser. The idea is this: place the mouse cursor in front of the first line of the text part of the page, hold down the left mouse button and, without releasing it, press the right Ctrl. The mouse will automatically be marked to the right, then the text will be scrolled down to the end, while standing out. Then the mouse stops in the middle of the screen, and it remains to clarify the selection using its position before releasing the left mouse button. If you press the right button before that, the process will be canceled, as the tultip in the center of the screen will report. Otherwise, automatic copying will occur, you will need to click the link to the next page below and repeat it there again. The buffer capture mode is enabled, so if necessary, you can copy small excerpts of the text in the usual way, and they will be added to the list of fragments.

The efficiency depends heavily on the amount of garbage at the bottom of the page, but even with its abundance it is still noticeably higher than with manual selection. Works in browsers based on Mozilla Firefox and Google Chrome, as well as in Opera Presto.

Use of other languages, coding and portability

"From the box" CLCL_AHK works with Autohotkey and AutoIt while ensuring complete portability. If you want to use it with other languages, the program will create virtual scripts specified in the extension header and try to execute them if the system knows what to do with such files. If not, in the CLCL.ahk settings, you can write the path to the executable file of the package. There you can also specify the required script encoding by specifying its number from here Code Page Identifiers (docs.microsoft.com), if the default 1251 does not suit you.

Display current clipboard status

Changes are displayed above the taskbar if the active application is not in full screen mode. In CLCL.ahk, you can customize the appearance, add the excluded applications or turn off this function altogether.

Automatic backup

It is produced at each start and at a given time interval. The program settings, templates and CLCL.ahk are copied. In the settings, you can change the interval and enable the clipboard history in the backup, if necessary, or turn it off altogether. Templates from the file regist.dat can be imported into a separate template folder from the menu File - Import, and from there to transfer where necessary. If you want to keep some backup for the future so that it is not deleted due to the limitation of their number, rename its folder so that it does not start with numbers, for example, adding a line at the beginning of the name.

FAQ

AT. How to change the key Alt + Q to another?

ABOUT. Change in program settings and in CLCL.ahk, then restart the program.

AT. How to change the key Ctrl + Shift + F11 to another?

ABOUT. Change first at CLCL.ahk. Then restart the program and only then change in the CLCL settings.

AT. Why do scripts often fall into the history of the buffer despite the option "Do not add samples to the log"?

ABOUT. This is a question for the author of the program...

AT. Is it possible to replace the old, time-tested CLCL, one that is part of the program?

ABOUT. It is possible, but in the latest versions the settings were moved to the root folder and the autoback will not work.

AT. CLCL has glitches with hotkey registration and swears loudly at startup...

ABOUT. The program has no glitches, except with keys of the form Ctrl + Shift + Alt +..., confused with keys with two modifiers, and which should not be used at all. If problems arise, you have to turn them off one by one, so it’s better to restart the program after each new appointment.
